# Presentation as Code — 中文使用说明

## 这是什么

一套用 AI 协作创建高质量交互式 HTML 演示文稿的方法论。

从"法务 AI 训练营"项目中提炼而来——5个模块、35+页、面向零编程背景的法务专业人员。经历了 Reveal.js 翻车、暗色主题否决、叙事结构重构等多轮迭代后沉淀的最佳实践。

## 核心理念

**先讲故事，再做设计，最后加交互。**

不要从"这页放什么图"开始，要从"观众看完应该记住什么"开始。

## 五阶段工作流

| 阶段 | 做什么 | 关键产出 |
|------|--------|---------|
| 1. 叙事架构 | 定义受众、设计故事弧线、每模块一个"记住这一点" | 模块大纲 |
| 2. 设计系统 | CSS 变量、组件库、字体、配色 | style.css |
| 3. 内容构建 | 每页一个 section，数据可视化优先于文字 | 各模块 HTML |
| 4. 交互层 | Chart.js、SVG 动画、手机/终端 mockup、现场 demo | 增强体验 |
| 5. 压力测试 | 离线测试、键盘导航、移动端、"奶奶测试" | 定稿 |

## 我们踩过的坑

- **Reveal.js 单文件** → CDN 挂了就白屏，71KB 难维护 → 改为每模块独立 HTML
- **暗色主题** → 非技术人员觉得"太程序员" → 白底 + 彩色点缀
- **纯文字幻灯片** → 观众提前读完就不听你说了 → 数据图表 + 一句话结论
- **"什么是AI"开场** → 无聊且居高临下 → 改为"30人+AI = 3000人产出"数据冲击

## 叙事模板

```
开场：为什么你现在就该关心（用数据制造紧迫感）
  ↓
去魅：它其实没那么神秘（用日常类比解释原理）
  ↓
工具：你能用的武器有哪些（现场演示 > 截图 > 文字）
  ↓
升级：驾驭它的思维方式（框架和方法论）
  ↓
行动：我自己是怎么做的（个人实践 > 抽象展望）
```

## 技术架构

```
presentation-name/
  00-opening.html      # 每个模块独立，可单独打开
  01-module-a.html
  02-module-b.html
  style.css            # 共享设计系统（CSS 变量 + 组件类）
  slides.js            # 共享导航（scroll-snap + 键盘 + 进度条）
```

**关键约束：**
- 零 CDN 依赖（离线可用）
- 零构建步骤（双击 HTML 就能看）
- 每页有且只有一个渐变标题
- 每页必须有 highlight-box（关键结论）

## 文件说明

| 文件 | 用途 |
|------|------|
| `SKILL.md` | 方法论主体，Claude 自动读取并遵循 |
| `design-system-reference.md` | 组件代码速查（HTML/CSS 片段，写代码时参考） |
| `README-zh.md` | 本文件，给人类看的中文说明 |

## 怎么用

### 自己用
文件已在 `~/.claude/skills/presentation-as-code/`，Claude Code 会自动发现。对话中说"帮我做个PPT"或"create slides"即可触发。

### 分享给同事
把整个 `presentation-as-code/` 目录拷贝到对方的 `~/.claude/skills/` 下即可。
