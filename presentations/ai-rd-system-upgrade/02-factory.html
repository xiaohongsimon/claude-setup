要建什么

三样核心基建

Spec范式（SDD-RIPER）

代码可消耗、可重生成，文档才是核心资产

类比：建筑图纸，而非砖头

契约中心

跨系统调用有据可依，AI不用猜接口

类比：公路上的交通规则

三道门禁体系

代码洪流中自动化拦截缺陷

类比：工厂的质检流水线

三层Spec

三层Spec体系

意图层
Requirement Spec
定义验收标准（AC），消除自然语言歧义
AI理解"为什么做"
契约层
Interface Spec
定义JSON Schema，技术隔离墙
编码Agent介入前必须Sign-off
施工图层
Implementation Spec
利用AST预定义技术路径
防止AI盲目修改存量逻辑
为什么Spec这么重要：有了Spec，代码坏了可以基于Spec重新生成；没有Spec，代码坏了只能人肉考古。Spec让代码从"不可替代"变成"可再生"。

上线
门禁 检查内容 通过标准 耗时目标
① Spec审查 需求完整性、AC可测试性、技术方案可行性 三层Spec完整 + 架构师Sign-off <4小时
② 架构约束 语法正确性、分层依赖、接口Schema一致 Harness规则全通过 <10分钟
③ 测试验收 增量覆盖率、契约测试、字段偏差检测 行覆盖率达标 + 契约0偏差 <30分钟
方向不对，后面再严也没用。门禁1是"方向判断"，必须人工+AI共同决策；门禁2、3逐步自动化，释放人工带宽。
红线：门禁3的"字段偏差阻断"不可降级，防止数据事故。
